Certainly! Here's a cheatsheet in a table format for various notations used in JavaScript:

| Notation | Description | Example |
| --- | --- | --- |
| 0b | Binary Literal | const binaryNumber = 0b1010; |
| 0o | Octal Literal | const octalNumber = 0o10; |
| 0x | Hexadecimal Literal | const hexNumber = 0x1F; |
| n or B | BigInt Literal (Suffix) | const bigIntNumber = 1234567890123456789012345678901234567890n; |
| BigInt() | BigInt Constructor | const bigIntNumber = BigInt("1234567890123456789012345678901234567890"); |
| b | BigInt Literal (Suffix for Bitwise Ops) | const bigIntNumber = 0b1010n; |
| Bitwise Ops | Bitwise Operations with BigInts | const result = bigInt1 & bigInt2; |

<Accordion title="Bitwise Operators">
    | Operator | Description | Example | Decimal Example Result |
    | --- | --- | --- | --- |
    | & | Bitwise AND | 5 & 3 | 1 |
    | ` | ` | Bitwise OR | 5 | 3 |
    | ^ | Bitwise XOR (Exclusive OR) | 5 ^ 3 | 6 |
    | ~ | Bitwise NOT (Complement) | ~5 | -6 |
    | \<\< | Left Shift | 5 \<\< 1 | 10 |
    | >> | Right Shift (Sign-propagating) | 5 >> 1 | 2 |
    | >>> | Right Shift (Zero-fill) | 5 >>> 1 | 2 |
    - **Bitwise AND (`&`):** Sets each bit to 1 if both bits are 1.
    - **Bitwise OR (`|`):** Sets each bit to 1 if either or both bits are 1.
    - **Bitwise XOR (`^`):** Sets each bit to 1 if only one of the bits is 1.
    - **Bitwise NOT (`~`):** Inverts all the bits.
    - **Left Shift (`<<`):** Shifts left by pushing zeros in from the right and letting the leftmost bits fall off.
    - **Right Shift (`>>`):** Shifts right, preserving the sign bit (copies the leftmost bit).
    - **Right Shift (`>>>`):** Shifts right, filling with zeros from the left.
